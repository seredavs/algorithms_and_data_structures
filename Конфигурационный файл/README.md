# C. Конфигурационный файл

## Условие

Вадим разрабатывает парсер конфигурационных файлов. Файл состоит из блоков, выделенных символами `{` и `}`. Блоки могут вкладываться друг в друга.

В файле встречаются переменные (имена состоят из строчных латинских букв, не более 10 символов). Изначально все переменные равны `0`.

Есть два типа присваиваний:
1. `<variable>=<number>` — присвоить переменной числовое значение
2. `<variable1>=<variable2>` — присвоить переменной значение другой переменной

**Важное правило:** новое значение переменной действует **до конца текущего блока**, после чего восстанавливается старое значение. Если внутри блока есть вложенные блоки, в них тоже действует новое значение (если они идут после присваивания).

Нужно для каждой строки вида `<variable1>=<variable2>` вывести значение, которое было присвоено.

---

## Формат ввода

Входные данные содержат от 1 до 10^5 строк. Каждая строка имеет один из четырех типов:
- `{` — начало блока
- `}` — конец блока  
- `<variable>=<number>` — присваивание числового значения
- `<variable1>=<variable2>` — присваивание значения переменной

Гарантируется:
- Ввод корректен
- Имена переменных — не более 10 строчных латинских букв
- Числа по модулю не превосходят 10^9
- Переменные могут совпадать (`a=a`)
- Ввод не содержит пробелов

## Формат вывода

Для каждой строки типа `<variable1>=<variable2>` выведите присвоенное значение (число) в отдельной строке.

---

## Пример

**Ввод:**
a=b
b=123
var=b
b=-34
{
c=b
b=1000000000
d=b
{
a=b
e=var
}
}
b=b


**Вывод:**
0
123
-34
1000000000
1000000000
123
-34
